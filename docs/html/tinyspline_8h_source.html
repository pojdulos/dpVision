506/******************************************************************************
507* *
508* Transformation functions *
509* *
510* TinySpline is a library focusing on transformations. That is, most *
511* functions are used to transform splines by modifying their state, e.g., *
512* their number of control points, their degree, and so on. Accordingly, each *
513* transformation functions specifies an input and output parameter (along *
514* with the other parameters required to calculate the actual transformation). *
515* By passing a different pointer to the output parameter, the transformation *
516* result is calculated and stored without changing the state of the input. *
517* This is in particular useful when dealing with errors as the original state *
518* will never be modified. For instance, let's have a look at the following *
519* code snippet: *
520* *
521* tsBSpline in = ... // an arbitrary spline *
522* tsBSpline out; // result of transformation *
523* *
524* // Subdivide 'in' into sequence of bezier curves and store the result *
525* // in 'out'. Does not change 'in' in any way. *
526* tsError err = ts_bspline_to_beziers(&in, &out); *
527* if (err != TS_SUCCESS) { *
528* // fortunately, 'in' has not been changed *
529* } *
530* *
531* Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
532* changed allowing you to handle the error properly. *
533* *
534* Unless stated otherwise, the order of the parameters for transformation *
535* functions is: *
536* *
537* function(input, [additional_input], output, [additional_output]) *
538* *
539* 'additional_input' are parameters required to calculate the actual *
540* transformation. 'additional_output' are parameters storing further result. *
541* *
542* Note: None of TinySpline's transformation functions frees the memory of the *
543* output parameter. Thus, when using the same output parameter multiple *
544* times, make sure to free memory before each call. Otherwise, you will *
545* have a bad time with memory leaks: *
546* *
547* tsBSpline in = ... // an arbitrary spline *
548* tsBSpline out; // result of transformations *
549* *
550* ts_bspline_to_beziers(&in, &out); // first transformation *
551* ... // some code *
552* ts_bspline_free(&out); // avoid memory leak. *
553* ts_bspline_buckle(&in, &out); // next transformation *
554* *
555* If you want to modify your input directly without having a separate output, *
556* pass it as input and output at once: *
557* *
558* tsBSpline s = ... // an arbitrary spline *
559* tsReal *knots = ... // a knot vector *
560* *
561* ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
562* *
563* Note: If a transformation function fails *and* input != output, all fields *
564* of the output parameter are set to 0/NULL. If input == output, your *
565* input may have an invalid state in case of errors. *
566* *
567******************************************************************************/
